Voy a ser sincero contigo. Soy desarrollador de software, probablemente tu tambien. Me encanta el software de codigo abierto. Estoy escribiendo esto en una laptop con Ubuntu usando el navegador Firefox. Ambas piezas de software son de codigo abierto, pero honestamente casi todo el software que usamos esta impulsado por codigo abierto de una forma u otra.

Beekeeper Studio no es diferente. Una porcion significativa del codigo fuente de Beekeeper Studio esta [disponible en GitHub](https://github.com/beekeeper-studio/beekeeper-studio) bajo la licencia GPLv3 (que ayuda a mantener todos los forks tambien como codigo abierto).

Como parte de mi compromiso con el codigo abierto, proporciono una edicion comunitaria de Beekeeper Studio que esta compuesta enteramente de codigo abierto. Tiene menos caracteristicas, pero para un estudiante o alguien que recien comienza con bases de datos, es perfecta.

Puedes descargar la edicion comunitaria en nuestra [pagina de releases en GitHub](https://github.com/beekeeper-studio/beekeeper-studio/releases/latest).

## Disponibilidad del codigo de componentes comerciales

Espero en el futuro hacer que todo el codigo de Beekeeper Studio este disponible publicamente en GitHub, con el codigo comercial siendo [source available](https://en.wikipedia.org/wiki/Source-available_software).

## Beekeeper Studio comenzo como un proyecto de codigo abierto

Debo el exito de Beekeeper Studio al codigo abierto y a la comunidad de codigo abierto. Inicialmente toda la aplicacion estaba disponible bajo la licencia GPL.

A medida que el proyecto ha crecido, mantener un proyecto tan grande sin ningun ingreso se volvio imposible, y asi tome la decision de crear un fork comercial.

Mantengo una generosa edicion comunitaria porque el codigo abierto es muy importante para mi, y no quiero dejar a mis usuarios gratuitos en la estacada. Si pudiera mantenerme con una aplicacion completamente de codigo abierto y gratuita, lo haria, pero desafortunadamente eso no es posible.

Todas las compras de Beekeeper Studio apoyan la inversion continua en la edicion comunitaria de codigo abierto.
